## 🔍 Core Concepts

### 🛡 Defender XDR = Unified Security Platform

Combines:
- Endpoint security (MDE)  
- Identity signals (Entra ID)  
- Email protection  
- Cloud app protection  

---

### 🚨 Incident vs Alert

- Alert → Single suspicious activity  
- Incident → Group of related alerts  

✅ Goal:
Investigate incidents, not just alerts  

---

### 🔄 SOC Workflow

Detect → Investigate → Contain → Remediate → Recover  

---

## ⚙️ Key Investigation Flow

Step 1 > Review incident  
Step 2 > Check alert evidence  
Step 3 > Analyse timeline  
Step 4 > Identify attacker activity  
Step 5 > Take action (isolate device / stop process)  
Step 6 > Confirm remediation  

---

## 🧠 Real-World Security Thinking

- Assume compromise  
- Investigate beyond alerts  
- Correlate multiple signals  
- Prioritise high-risk incidents  
- Automate where possible  

---

## 🔐 Common Response Actions

- Isolate device  
- Run antivirus scan  
- Stop process  
- Add indicator (block file/IP)  
- Collect investigation package  

---

## 🚀 Key Takeaways

- Defender XDR is a **central SOC platform**  
- Focus on **incident-level investigation**  
- Automation improves response speed  
- Threat hunting (KQL) is a critical skill  
- Endpoint visibility is essential
